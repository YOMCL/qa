User must configure GitHub Actions secrets and trigger a workflow_dispatch run.

Plan 01 created `.github/workflows/app-maestro.yml`. It references 5 GitHub Actions secrets:

**Required secrets (workflow fails without these):**
1. `APK_URL` — Download URL for the YOM Ventas APK (`me.youorder.yomventas`). Options: S3 pre-signed URL, Firebase App Distribution link, or any HTTPS URL that returns the APK binary.
2. `APP_PACKAGE` — Value: `me.youorder.yomventas`
3. `TEST_SELLER_EMAIL` — Seller email for Prinorte staging login
4. `TEST_SELLER_PASSWORD` — Seller password for Prinorte staging login

**Optional secret (fallback to GitHub issue if not set):**
5. `SLACK_MAESTRO_WEBHOOK` — Incoming webhook URL for a Slack channel (e.g., `#qa-alerts`)

Steps:
1. Go to GitHub repo > Settings > Secrets and variables > Actions
2. Add the 4 required secrets
3. (Optional) Add SLACK_MAESTRO_WEBHOOK
4. Go to Actions tab > "Maestro Nightly — Prinorte APP" > "Run workflow"
5. Watch the run — key checkpoints:
   - "Enable KVM" completes in < 10 seconds
   - "Download APK" shows file size
   - "Run Maestro on Emulator" boots emulator and runs flows
   - "Upload artifacts" creates `maestro-report-*` artifact
  </action>
